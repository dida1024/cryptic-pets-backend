## 📋 产品概述

### 产品定位

Cryptic Pets 是一个专为爬宠玩家和繁殖户设计的**饲养管理SaaS平台**，聚焦玉米蛇饲养场景，通过移动端优先的设计理念，解决用户在日常饲养中遇到的**记录混乱、数据丢失、喂食遗忘**等核心痛点。

**我们解决的问题**：
- ❌ 纸笔记录容易遗失，Excel表格协作混乱
- ❌ 忘记上次喂食时间，导致过度喂养或营养不足
- ❌ 无法直观看到体重增长曲线，难以及时发现健康问题
- ❌ 繁殖户管理几十条蛇，批量记录效率低下
- ❌ 基因配对复杂，容易出现近亲繁殖

### 产品愿景

**短期愿景（1年内）**：成为中国最受爬宠玩家喜爱的饲养记录工具，服务 1,000+ 付费用户

**中期愿景（3年内）**：扩展到球蟒、守宫等常见爬宠品种，覆盖 50% 的中国爬宠玩家市场

**长期愿景（5年+）**：打造全球领先的爬宠饲养管理生态平台，集成繁殖管理、社区交流、交易市场等功能

### 核心价值主张

**为什么用户应该选择我们？**

1. **3秒记录，比纸笔快10倍**  
   移动端拍照 → 选择宠物 → 一键保存，喂食后立即记录，不再遗忘

2. **智能提醒，从不漏喂**  
   基于历史数据自动计算喂食周期，提前1天推送通知，避免喂食失误

3. **成长可视化，一目了然**  
   自动生成体重增长曲线，及时发现生长异常或健康问题，优化饲养策略

4. **数据自主可控**  
   支持自部署或云托管，数据永远属于用户，保护商业机密和隐私

5. **专为爬宠设计**  
   深度理解玉米蛇饲养场景（喂食周期、蜕皮记录、基因管理），而非通用宠物工具的简单移植

### 用户优先级策略

基于市场分析和产品能力，我们聚焦以下用户优先级：

| 优先级 | 用户类型 | 占比 | 特征 | 付费能力 | 获客难度 | 策略 |

| **P0** | 个人玩家（1-10条） | 75% | 初次饲养或轻度玩家 | ¥100-300/年 | 中 | **主力用户**，优先优化移动端体验 |
| **P1** | 自繁玩家（10-50条） | 20% | 同时管理十几到几十条蛇 | ¥300-600/年 | 低 | **高价值用户**，提供批量操作 |

**为什么聚焦个人玩家（P0）？**
1. 市场规模最大（75%），容易形成规模效应
2. 移动端场景明确（喂食后立即记录），产品价值清晰
3. 口碑传播能力强（爬宠玩家社群活跃）
4. 付费意愿合理（¥99-199/年可接受）

### 主要用户画像

#### 用户类型1：个人玩家（P0 - 主力用户）

**张同学，22岁，大学生，饲养1条玉米蛇**

- **特征**：  
  - 初次饲养，在校大学生或职场新人
  - 通过小红书/B站了解爬宠，冲动购买
  - 有一定技术素养，熟悉App操作
  - 喜欢分享宠物照片到社交平台

- **核心需求痛点**：  
  - ❌ **忘记喂食时间**：不清楚玉米蛇喂食周期（7-10天），担心喂多或喂少
  - ❌ **无法看到成长变化**：体重记录在纸上，看不出增长趋势
  - ❌ **拒食时不知道原因**：不知道是蜕皮前正常现象还是健康问题
  - ❌ **记录麻烦**：纸笔记录容易丢，Excel打开太慢

- **使用场景**：  
  1. 每周喂食后，用手机拍照记录
  2. 每月称重，查看体重曲线
  3. 接收喂食提醒通知
  4. 蜕皮前拒食时，查看历史记录判断是否正常

- **付费意愿**：¥99-199/年（相当于1-2袋鼠粮的价格）

#### 用户类型2：自繁玩家（P1 - 高价值用户）

**李先生，35岁，爬宠工作室主理人，饲养30条玉米蛇**

  - 从个人爱好发展为小型生意

  1. 喂食日当天，批量添加30条蛇的喂食记录

  3. 配对前，查询基因库预测后代形态
  4. 为售出的宠物生成PDF档案报告

  - ❌ **团队协作**：多名员工需要权限管理和操作审计

  1. 自部署私有实例，确保数据安全
  2. 配对前系统自动检测近亲风险
  3. 生成基因溯源报告，为高端客户提供血统证明
  4. 团队成员协同录入数据，管理员审计操作日志

### MVP功能详解（v1.0 - 必须实现）

#### 1. 移动端App（P0 - 核心）
**为用户提供随时随地记录的能力，喂食后立即拍照保存**

- **核心能力**：
  - 📱 **快速记录**：打开App → 拍照 → 选择宠物 → 一键保存（< 5秒）
  - 📷 **图片管理**：自动压缩上传，支持相册选择或即时拍照
  - 🔔 **推送通知**：喂食提醒、称重提醒推送到手机
  - 📊 **数据浏览**：查看宠物列表、历史记录、体重曲线
  - 🔄 **离线支持**：断网时可记录，联网后自动同步（v1.5考虑）

- **业务价值**：  
  - 解决用户"记录麻烦"的核心痛点
  - 移动端场景匹配度高（喂食场景在手机边）
  - 拍照功能增强记录的乐趣和完整性

- **技术方案**：
  - **技术栈**：Flutter（跨平台，iOS + Android）或 React Native
  - **认证方式**：JWT Token，支持自动续期
  - **图片上传**：压缩到 < 500KB，上传到对象存储（阿里云OSS/腾讯云COS）
  - **推送服务**：Firebase Cloud Messaging（FCM）或极光推送

- **用户故事**：
  ```
  作为 个人玩家
  我想 在喂完蛇后立即用手机记录
  以便 不会忘记记录，养成科学饲养习惯
  
  验收标准：
  - 从打开App到完成记录 < 10秒
  - 支持拍照或相册选择
  - 自动填充当前日期时间
  - 如果只有1条宠物，自动选中
  ```

---

#### 2. Web管理后台（P0 - 核心）
**为用户提供完整的数据管理和分析能力**

- **核心能力**：
  - 🖥️ **宠物档案管理**：创建、编辑、删除宠物信息（名称、性别、出生日期、照片）
  - 📝 **记录管理**：查看、编辑、删除历史记录，支持筛选和搜索
  - 📈 **数据可视化**：体重增长曲线图、喂食周期统计
  - ⚙️ **设置中心**：账号信息、提醒配置、数据导出
  - 👥 **用户系统**：注册、登录、密码管理、注销

- **业务价值**：
  - 满足用户在电脑端批量管理数据的需求
  - 提供比移动端更强大的数据分析能力
  - 方便繁殖户导出数据或生成报告

- **技术方案**：
  - **技术栈**：React 18+ / Vue 3 + TypeScript
  - **UI框架**：Ant Design / Element Plus（中文友好）
  - **图表库**：ECharts（中文生态好）或 Chart.js
  - **响应式设计**：支持桌面和平板，移动端体验优先使用App

- **用户故事**：
  ```
  作为 自繁玩家
  我想 在电脑上查看所有宠物的体重对比
  以便 筛选出体质好的个体用于繁殖
  
  验收标准：
  - 支持多条宠物体重曲线叠加对比
  - 可以按时间范围筛选（最近1个月/3个月/1年）
  - 可以导出图表为图片
  ```

---

#### 3. 喂食与称重记录（P0 - 核心）
**最基础但最重要的记录功能**

- **核心能力**：
  - **喂食记录**：

#### 4. 智能喂食提醒（P0 - 核心）
**基于历史数据自动计算喂食周期，推送提醒通知**

- **核心能力**：
  - **自动计算喂食周期**：
    - 分析最近3-5次喂食间隔，计算平均周期
    - 考虑季节因素（冬季喂食间隔更长）
    - 根据宠物年龄调整（幼体5-7天，成体10-14天）
  - **提醒推送**：
    - 在预计喂食日**前1天**推送提醒
    - 通知内容："明天该给【小白】喂食啦！上次喂食：7天前"
    - 用户可自定义提醒时间（如：每天晚上8点）
  - **提醒管理**：
    - 用户可以手动设置喂食周期（覆盖自动计算）
    - 记录后自动重置下次提醒时间
    - 支持暂停提醒（如：冬眠期）

- **业务价值**：
  - **核心价值主张之一**，解决"忘记喂食"的刚需
  - 提升用户活跃度（定期收到通知，回到App）
  - 增强产品粘性，用户依赖提醒功能

- **技术方案**：
  - **计算逻辑**：
    ```python
    # 伪代码
    recent_feedings = get_last_n_feedings(pet_id, n=5)
    intervals = calculate_intervals(recent_feedings)
    avg_interval = mean(intervals)
    next_feeding_date = last_feeding_date + avg_interval
    reminder_date = next_feeding_date - 1 day
    ```
  - **定时任务**：Celery + Redis，每天凌晨1点扫描需要提醒的宠物
  - **推送渠道**：
    - 移动端：FCM / APNs / 极光推送
    - 可选：Telegram Bot、邮件（v1.5）

- **用户故事**：
  ```
  作为 经常忘记喂食的新手玩家
  我想 系统自动提醒我该喂食了
  以便 不会因为忘记导致宠物营养不良
  
  验收标准：
  - 在预计喂食日前1天收到推送通知
  - 通知包含：宠物名称、上次喂食时间、建议食物大小
  - 记录喂食后，自动计算下次提醒时间
  - 可以在设置中关闭某条宠物的提醒
  ```

## 💼 使用场景

### 场景1：新手玩家日常饲养记录
**帮助初次饲养者养成科学记录的习惯，避免因遗忘造成的饲养问题**

**角色**：个人玩家（初次饲养玉米蛇）  
**目标**：定期记录喂食和称重数据，跟踪宠物健康状况  
**流程**：
1. 注册账号并登录平台
2. 创建宠物档案：上传照片、填写名称、性别、出生日期、品种信息
3. 每周喂食后记录：选择"喂食记录"，输入食物类型（如：冻乳鼠）、重量（10g）、是否进食
4. 每月称重：选择"称重记录"，输入体重数据（如：45g），查看体重增长曲线
5. 发现异常时添加健康记录：如拒食、呕吐等症状，记录处理方式
6. 查看历史记录，分析喂食周期是否合理

**预期结果**：
- 形成完整的饲养日志，不再遗忘上次喂食时间
- 通过体重曲线及时发现生长异常
- 积累经验，优化喂食间隔和食物大小

### 场景2：自繁玩家批量管理与繁殖计划
**高效管理多条宠物，记录不同批次的喂食周期，辅助繁殖决策**

**角色**：个人自繁玩家（同时管理 20-50 条玉米蛇）  
**目标**：批量管理多条宠物，追踪体质差异，规划繁殖配对  
**流程**：
1. 批量创建宠物档案，按批次或品系分组（如：2024年第一批、Amel het Anery等）
2. 喂食日当天，批量添加喂食记录，标注特殊情况（拒食、需要辅助进食等）
3. 定期称重，通过体重对比筛选出体质较好的个体
4. 为繁殖用蛇添加详细的基因信息（纯合/杂合、显性/隐性）
5. 查询基因库，模拟配对组合，预测后代形态
6. 记录配对、产卵、孵化等繁殖事件
7. 为售出的宠物转移所有权，交付给新主人

**预期结果**：
- 告别纸质记录，不再混淆不同批次的喂食周期
- 通过数据筛选优质种蛇，提高繁殖质量
- 科学配对，增加目标形态的孵化概率
- 展示专业的繁殖记录，提升客户信任度

6. 生成基因溯源报告，为高端客户提供血统证明
7. 分析不同品系的健康数据和繁殖成功率，淘汰劣质基因
8. 定期导出数据备份，确保数据安全

**预期结果**：
- 数字化管理庞大的宠物数量，不再依赖 Excel 或纸质记录
- 科学避免近亲繁殖，保持种群健康

- 为客户提供专业的血统证明，提升品牌价值
- 数据自主可控，保护商业机密

## 🏗️ 技术架构（简化版）

### 系统架构

Cryptic Pets 采用前后端分离的架构设计，后端基于领域驱动设计（DDD）的四层架构：

```

│                     前端应用层                           │
│            (Web App / Mobile App / 第三方集成)          │
└────────────────────┬────────────────────────────────────┘
                     │ HTTP/HTTPS (RESTful API)
                     ▼
┌─────────────────────────────────────────────────────────┐
│                  Interface Layer (接口层)                │
│  - HTTP 路由 (FastAPI Routers)                          │
│  - 请求/响应模型 (Pydantic Schemas)                      │
│  - 统一异常处理 & API 响应格式                           │
│  - JWT 认证中间件                                        │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│              Application Layer (应用层)                  │
│  - Command/Query Handlers (CQRS)                       │
│  - 业务流程编排                                          │
│  - 应用服务 (Use Cases)                                 │
│  - 视图模型 (View Models)                               │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│                Domain Layer (领域层)                     │
│  - 实体 (Entities): Pet, PetRecord, Breed, Gene        │

│  - 聚合根 (Aggregate Roots)                             │
│  - 领域事件 (Domain Events)                             │
│  - 仓储接口 (Repository Interfaces)                     │
└─────────────────────────────────────────────────────────┘
                     ▲
                     │
┌─────────────────────────────────────────────────────────┐
│            Infrastructure Layer (基础设施层)             │
│  - 数据库实现 (PostgreSQL + SQLModel)                   │

```

**架构特点**：
- **依赖倒置**：高层模块不依赖低层模块，都依赖于抽象
- **单向依赖**：Interface → Application → Domain ← Infrastructure
- **领域纯净**：Domain 层不依赖任何外部框架，保持业务逻辑纯粹

### 技术栈

#### 后端技术
- **框架**：FastAPI 0.111.0+（Python 3.12+）
- **数据库**：PostgreSQL 15+
- **ORM**：SQLModel 0.0.21+ (基于 SQLAlchemy 2.0)
- **数据验证**：Pydantic 2.8.2+
- **认证**：JWT (JSON Web Tokens)
- **密码加密**：bcrypt (via passlib)
- **日志**：Loguru（结构化日志）
- **监控**：Sentry（错误追踪）
- **代码质量**：Ruff（linting）、MyPy（类型检查）

#### 架构设计
- **设计模式**：领域驱动设计（DDD）
- **分层架构**：四层架构（Domain → Application → Infrastructure → Interface）
- **API设计**：RESTful API with OpenAPI 3.0
- **数据安全**：bcrypt 密码加密、JWT 认证、SQL 参数化查询
- **事件驱动**：领域事件 + 事件处理器
- **CQRS 轻量实现**：命令/查询分离

#### 部署与运维
- **容器化**：Docker + Docker Compose
- **反向代理**：Nginx (推荐)
- **进程管理**：Uvicorn (ASGI Server)
- **数据库连接池**：SQLAlchemy async pool

### 数据安全

- 🔒 **身份认证**：JWT Token 认证，支持 Token 过期和刷新机制
- 🔒 **密码保护**：bcrypt 加密存储，不可逆加密算法
- 🔒 **SQL 注入防护**：使用 SQLModel 的参数化查询，自动转义

- 🔒 **敏感数据脱敏**：日志中自动过滤密码、Token 等敏感信息
- 🔒 **HTTPS 支持**：生产环境强制 HTTPS 加密传输

## ⚠️ 风险评估与应对

### 产品风险

#### 风险1：MVP开发周期过长，错过推广窗口

**风险等级**：🔴 高

**描述**：移动端App开发需要3-6个月，如果延期可能错过小红书/B站的爬宠内容热度窗口期

**影响**：
- 竞品可能抢先推出类似产品
- 错过最佳推广时机，获客成本大幅增加
- 团队士气受挫，资金链紧张

**应对策略**：
1. **方案A - 渐进式发布**：
   - 先发布Web版（2个月），满足电脑端用户
   - 再发布PWA（渐进式Web应用），提供基础移动端体验
   - 最后发布原生App（3个月后）
2. **方案B - 技术选型优化**：
   - 使用Flutter跨平台开发，iOS+Android同时完成
   - 复用React组件库，减少前端开发时间
3. **方案C - 外包加速**：
   - 前端界面外包给专业团队，2个月交付
   - 核心后端和业务逻辑自己把控

**推荐方案**：方案A + 方案B组合，先上线Web版+PWA快速验证，同时开发原生App

#### 风险2：付费转化率低于预期（< 10%）

**风险等级**：🟡 中

**描述**：用户觉得免费版够用或Excel免费更好，不愿意付费¥99/年

**影响**：
- 第1年收入目标无法完成（预期¥189,350）
- 无法覆盖运营成本（¥226,200），资金链断裂
- 被迫调整商业模式或关闭项目

**应对策略**：
1. **优化免费版限制**：
   - 将宠物数量限制从3条降到1条（更严格）
   - 智能提醒功能完全锁定（核心价值）
   - 体重曲线只显示最近1个月（限制价值）
2. **增强付费版价值感知**：
   - 在免费版中高频展示付费功能的好处
   - 提供7天免费试用（先体验价值再付费）
   - 早鸟优惠：前100名用户5折（¥49/年）

   - 推出¥9.9/月的月付方案（虽然年付更划算）
   - 学生优惠¥69/年（30%折扣）
4. **增加付费触点**：
   - 用户添加第2条宠物时，弹窗升级提示
   - 第一次拒食/体重下降时，提示"付费版可智能分析异常"

**衡量标准**：如果3个月后转化率仍 < 10%，立即执行方案1+2

Cryptic Pets 的**所有功能都是开源的**，付费服务只是提供部署、运维和技术支持，不存在功能阉割。这是我们对开源精神的承诺。

**不会，数据永远是您的。**

- **代部署用户**：
  - 服务器归您所有，数据始终保留
  - 如果不续费技术支持，我们会停止运维服务，但**不会删除您的数据或关闭服务器**
  - 您可以随时导出数据或自行维护

### 术语表

为了帮助非专业用户理解产品中的专业术语，我们整理了常用术语的解释：

#### 爬宠相关术语
- **玉米蛇（Corn Snake）**：北美原产的一种常见宠物蛇，学名 Pantherophis guttatus，性格温顺，适合新手饲养
- **形态（Morph / Morphology）**：指玉米蛇的外观表现型，由基因组合决定，如 Amelanistic（白化）、Anerythristic（黑化）等
- **基因型（Genotype）**：生物体的遗传组成，决定其外观和特征
- **表现型（Phenotype）**：生物体的外观特征，是基因型的外在表现
- **纯合（Homozygous）**：某个基因位点上的两个等位基因相同，通常表示为 **het** 或记号如 AA
- **杂合（Heterozygous）**：某个基因位点上的两个等位基因不同，通常表示为 **het** 或记号如 Aa
- **隐性基因（Recessive）**：需要纯合状态才能表现的基因
- **显性基因（Dominant）**：只需一个等位基因即可表现的基因
- **共显性（Co-dominant）**：两个等位基因同时表现，产生中间表型
- **蜕皮（Shedding）**：爬行动物定期脱落旧皮的生理现象
- **拒食（Refusing Food）**：宠物蛇拒绝进食，可能因为蜕皮前、温度不适、疾病等原因

#### 技术术语
- **DDD（Domain-Driven Design）**：领域驱动设计，一种软件架构方法，强调业务领域建模
- **RESTful API**：一种 Web API 设计风格，使用 HTTP 方法（GET、POST、PUT、DELETE）操作资源
- **JWT（JSON Web Token）**：一种安全的身份认证方式，用于 API 鉴权
- **Docker**：容器化技术，将应用和依赖打包，实现快速部署和环境隔离
- **PostgreSQL**：一种开源的关系型数据库，功能强大，适合复杂数据管理
- **软删除（Soft Delete）**：数据标记为已删除但不物理删除，可恢复和追溯
- **异步 I/O（Async I/O）**：非阻塞的输入输出操作，提高并发性能
- **ORM（Object-Relational Mapping）**：对象关系映射，将数据库表映射为编程语言中的对象
- **OpenAPI / Swagger**：API 文档规范和工具，自动生成交互式 API 文档

#### 业务术语
- **自部署（Self-Hosted）**：用户在自己的服务器上部署系统，完全掌控数据
- **代部署（Managed Hosting）**：由服务商代为部署和维护系统，用户只需使用
- **SLA（Service Level Agreement）**：服务水平协议，约定服务可用性和响应时间
- **数据迁移（Data Migration）**：将数据从一个系统迁移到另一个系统
- **血统证明（Pedigree Certificate）**：记录宠物血统和基因信息的证明文件
